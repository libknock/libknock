# v0.1.0-rc3.3 release notes

rc3.3 is a focused lifecycle and documentation hardening pass on top of rc3.2. It keeps the SDK/API boundary stable and does not add new platform validation claims.

## Changes

- Added `gatewaycore.RevokeFirewallDetached` for timer and shutdown cleanup after serving contexts may already be cancelled.
- Switched gate and relay TTL timer firewall revokes to the detached helper while keeping normal request-path revoke operations bound to their caller context.
- Added regression coverage for detached revoke contexts at the helper layer and through both gate and relay timer paths.
- Clarified `MarkFirewall` renewal behavior: repeated valid knocks renew the firewall lease from the latest accepted knock, and earlier timers become stale when their lease id no longer matches.
- Documented that `knock-firewall-only` stores a wildcard session with port `0`, while the firewall lease remains bound to the real protected port.
- Updated README and gate/relay documentation to state that repeated valid knocks renew the firewall allow window.


## Known limits

No new real-host firewall or packet-capture validation is claimed by rc3.3. nftables/iptables/ipset real-host mutation, UDP passive DROP scan behavior, TCP SYN paths, Windows WinDivert/Npcap, macOS BPF/pcap, long fuzz, and production throughput remain target-host validation items unless separately recorded in `docs/validation/rc3.3.md` or a later validation record.
