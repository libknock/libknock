# v0.1.0-rc3.4 release notes

rc3.4 is a security and lifecycle hardening release candidate. It preserves the stable root SDK API and fixes audited correctness defects in compatibility relay wiring, authentication budgeting, replay-cache handling, firewall configuration, and bounded policy state.

## Security and correctness

- Serializes the legacy session-bound knock sender `SetSessionID` + `Knock` sequence so concurrent compatibility-client dials cannot cross-wire session identifiers.
- Makes relay shutdown close active and pending connections, so cancellation cannot remain blocked behind an active bidirectional copy.
- Adds context-aware relay copying with error propagation. Unexpected copy failures now emit `RelayError` rather than an incorrect successful relay event.
- Correctly budgets envelope-v2 `hint_mode=none` authentication as candidate count times eligible frame-size buckets.
- Rejects typed-nil replay caches safely and creates a default replay cache only through the owning server constructor.
- Makes nftables honor `enable_ipv6: false`; IPv6 rules and sets are omitted and IPv6 allow requests are rejected.
- Enables strict YAML field validation, rejects conflicting `secret` and `secret_file`, rejects negative numeric values, and changes the compatibility command default knock transport to UDP.
- Stops UDP listener context watchers when serving exits unexpectedly.
- Makes bounded ban lists preserve active bans rather than silently evicting them; callers that need a result can use `BanE` or `BanUntilE` and handle `ErrBanListFull`.
- Declares `gate.Gate` single-lifecycle: reuse returns `ErrGateStarted`, and starting after close returns `ErrGateClosed`.

## Release consistency

- Aligns nested Prometheus and gRPC modules with `github.com/libknock/libknock v0.1.0-rc3.4` while retaining local source-tree replacements.

## Validation

The rc3.4 local gate ran:

```bash
scripts/check.sh
scripts/check-integration.sh
scripts/check-api.sh
scripts/release-check.sh
go test -race ./relay
scripts/package-release.sh --with-vendor v0.1.0-rc3.4 <dist>
sha256sum -c <dist>/*.sha256
```

Automated validation includes unit, race, short fuzz, benchmark smoke, nested-module, example, vendored-release, API-snapshot, documentation-link, and archive checks. It does not establish real-host nftables/iptables/ipset rule behavior, passive packet capture behavior, TCP SYN packet paths, Windows/macOS packet-driver behavior, long fuzzing, or production throughput characteristics.
