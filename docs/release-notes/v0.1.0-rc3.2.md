# v0.1.0-rc3.2 release notes

rc3.2 is a CI recovery, release-process, observability/config-guard, code-quality, documentation, and coding-agent integration hardening pass on top of rc3.1. It keeps the SDK/API boundary stable and does not add new platform validation claims.

## Changes

- Fixed the GitHub Actions/vendor mismatch by committing the workspace vendor tree used by CI and release archive validation.
- Made `scripts/check-duplication.sh` fail hard in `STRICT=1` / release-check mode when `dupl` is unavailable, while preserving warning-only behavior for ordinary development scans.
- Added a non-git source-tree fallback to `scripts/package-release.sh`; git checkouts still package `HEAD` with `git archive`.
- Extracted shared iptables cleanup code used by the plain `iptables` and `ipset-iptables` backends, with tests covering cleanup command behavior.
- Added `netx.ErrAuthBackpressure` / auth-drop observability for full pending queues and mapped it into Prometheus dropped/pending counters.
- Added early validation for built-in static/rotating secret resolvers using EnvelopeV2 `HintModeNone` with too many candidates; large client populations should use route hints.
- Clarified `TTLLRU.Len`, `ActiveLen`, and `LenAfterSweep` semantics with tests.
- Added tests protecting public auth-error labeling, `NewGateway` config-field copying, and firewall missing-object string matching.
- Corrected and clarified the README knock-method support table so every row has consistent columns.
- Added `docs/llms.md`, IDE LLM scaffolding, agent recipes, task-matrix entries, and rc3.2 validation/release documentation.
- Refreshed platform-support, known-limitations, validation, release-checklist, and observability docs to avoid overclaiming real-host firewall/packet validation.

## Validation commands

The rc3.2 release gate should include:

```sh
go test -mod=vendor ./...
go vet -mod=vendor ./...
go build -mod=vendor ./...
go test -race -mod=vendor ./auth ./firewall ./gate ./knock ./netx ./policy ./protocol ./relay ./internal/cache ./internal/gatewaycore
go test -mod=vendor ./observability/prometheus/...
go test -mod=vendor ./test/integration/grpc/...
go test -mod=vendor ./examples/grpc-client/... ./examples/grpc-server/...
scripts/check-api.sh
scripts/check-doc-links.py
STRICT=1 scripts/check-duplication.sh .
scripts/check.sh
scripts/release-check.sh
scripts/package-release.sh --with-vendor v0.1.0-rc3.2 dist/
sha256sum -c dist/*.sha256
```

## Known limits

No new real-host firewall or packet-capture validation is claimed by rc3.2. nftables/iptables/ipset real-host mutation, UDP passive DROP scan behavior, TCP SYN paths, Windows WinDivert/Npcap, macOS BPF/pcap, long fuzz, and production throughput remain target-host validation items unless separately recorded in `docs/validation/rc3.2.md` or a later validation record.

## Artifacts

Publish both:

- `libknock-v0.1.0-rc3.2.zip`
- `libknock-v0.1.0-rc3.2-with-vendor.zip`

Each archive should have a matching `.sha256` file. The standard archive excludes `vendor/`; the `with-vendor` archive includes `vendor/`, `vendor/modules.txt`, `go.work`, and `go.work.sum`.
