# v0.1.0-rc3.1 release notes

rc3.1 is a release-candidate hardening pass on top of rc3. It keeps the SDK boundary stable and focuses on lifecycle safety, public API edge cases, structured session keying, crypto test anchors, metrics configuration validation, and mainline metadata consistency.

## Changes

- Replaced raw/passive listener context-close double-close patterns with a shared once-closed file-descriptor helper for Linux AF_PACKET and macOS BPF listeners.
- Added `netx.ErrNilListener`, `netx.NewListener`, root `ErrNilListener`, and nil-listener tests so public listener wrapping fails explicitly instead of constructing a panic-prone listener.
- Migrated relay knock-session cache keys from delimiter-joined strings to structured `{remote, client_id, port}` keys while preserving port/client/remote isolation and connection-count behavior.
- Strengthened replay-cache capacity tests, including concurrent fail-closed behavior at the configured cap.
- Added `internal/cryptox` HKDF-SHA256 RFC vector coverage, stable HMAC truncation vectors, constant-time equality checks, and invariant documentation for panic-shaped helpers.
- Added Prometheus adapter option validation for relay duration buckets and documented `New` versus `MustNew` startup failure semantics.
- Updated active nested modules to the current pre-release line with local `replace` directives, regenerated workspace vendor metadata, and kept current-mainline docs free of stale hardcoded package-version claims.

## Known limits

No new real-host firewall or packet-capture validation is claimed by rc3.1. nftables/iptables/ipset real-host mutation, UDP passive DROP scan behavior, Windows WinDivert/Npcap, macOS BPF/pcap, long fuzz, and production throughput remain target-host validation items.
