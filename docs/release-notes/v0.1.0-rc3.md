# v0.1.0-rc3 release notes

rc3 is a release-candidate hardening pass focused on observability, API freeze precision, LLM integration guardrails, and conservative validation boundaries. It does not claim new real-host firewall or packet-capture coverage.

## Changes

- Added replay-cache full observability: `OnReplayCacheFull`, Prometheus replay cache capacity gauges, and full-cache rejection counter.
- Added relay pending-queue pressure observability: pending-full counter, dropped counter, and last observed pending depth.
- Added `relay.Config`, `Config.WithDefaults()`, and `relay.NewGateway(cfg)` as the preferred constructor path while keeping public `relay.Gateway` fields compatible for v0.1.x.
- Strengthened API freeze checks with a signature-level root API snapshot covering function signatures, exported type shapes, and const/var declarations.
- Made duplication scanning release-safe in offline `with-vendor` environments by treating external `dupl` absence as a warning instead of a hard dependency.
- Documented synchronous `OnAuthenticated` callback constraints and bounded-queue async patterns.
- Added machine-readable agent task matrix and Chinese agent/anti-pattern summaries for LLM-assisted changes.
- Clarified UDP active vs passive scan behavior, iptables cleanup risk, protocol legacy compatibility, and XChaCha20-Poly1305 terminology.
- Added rc3 validation record and formal not-run table for environment-limited platform checks.

## Validation commands

```sh
go test ./...
go vet ./...
go test -race ./auth ./firewall ./gate ./knock ./netx ./policy ./protocol ./relay
go -C observability/prometheus test ./...
go -C observability/prometheus vet ./...
scripts/check-api.sh
scripts/release-check.sh
scripts/package-release.sh --with-vendor v0.1.0-rc3 dist/
```

## Known limits

See `docs/validation/rc3.md`. nftables/iptables/ipset real-host mutation, UDP passive DROP scan behavior, Windows WinDivert/Npcap, macOS BPF/pcap, long fuzz, and production throughput remain not validated in this release environment.
