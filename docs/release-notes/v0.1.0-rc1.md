# v0.1.0-rc1 release notes

`libknock` v0.1.0-rc1 is a release candidate for an embeddable pre-application TCP authentication SDK.

## Stable SDK surface

- Root package TCP auth helpers: `WrapListener`, `NewServer`, `ServerAuth`, `ClientAuth`, `Dialer`.
- Auth frame v1 compatibility path.
- TCP auth envelope v2 release-candidate path.
- Listener-owned replay cache lifecycle through `WrapListener` / `NewServer`.

## Advanced and platform-specific paths

- Gate and relay live in subpackages.
- Firewall-backed modes require target-host validation.
- UDP passive and TCP SYN methods require platform packet capture/raw socket support.
- `cmd/knock-proxy` is a compatibility command, not the full historical product.

## Breaking changes

- UDP JSON knock packets are removed. UDP knock now requires binary AEAD frames.
- No JSON fallback is provided.
- New clients and servers must agree on binary UDP knock framing.
- Legacy JSON fields are rejected only for knock-frame compatibility settings; unrelated JSON output/logging settings remain valid.
- Root package APIs are narrowed; advanced knobs are available through subpackages.

## Not hardware validated by repository automation

- Linux nftables / iptables / ipset-iptables real host enforcement.
- UDP passive DROP behavior.
- Windows WinDivert / Npcap paths.
- macOS BPF / pcap paths.
- Long-duration fuzzing and production performance characterization.
