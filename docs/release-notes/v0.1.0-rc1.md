# v0.1.0-rc1 release notes

`libknock` v0.1.0-rc1 is a GitHub pre-release candidate for an embeddable pre-application TCP authentication SDK.

## Stable SDK surface

- Root package TCP auth helpers: `WrapListener`, `NewServer`, `ServerAuth`, `ClientAuth`, `Dialer`.
- Root package aliases for stable constants and extension interfaces: `MinSecretSize`, `SecretResolver`, `SecretCandidate`, `ReplayCache`, `KnockSender`, `SessionBoundKnockSender`, `KnockSessionStore`, `EventSink`, `Policy`, `FrameMeta`, and `PeerIdentity`.
- Auth frame v1 compatibility path.
- TCP auth envelope v2 release-candidate path.
- Listener-owned replay cache lifecycle through `WrapListener` / `NewServer`.

## Security and compatibility hardening

- Race-safe replay and ban-list cache handling.
- Nil connection guards on public auth entry points via `auth.ErrNilConn`.
- Mandatory knock-session binding for `knock-auth-only` and `knock-firewall-auth` paths.
- Deterministic no-hint envelope v2 candidate ordering and explicit `auth.ErrTooManyCandidates` overflow behavior.
- Strict envelope v2 bucket validation for the supported set `128 / 192 / 256 / 384 / 512`.
- Legacy TCP SYN sequence namespace compatibility is opt-in.

## Advanced and platform-specific paths

- Gate and relay live in subpackages.
- Firewall-backed modes require target-host validation.
- UDP passive and TCP SYN methods require platform packet capture/raw socket support.
- `protocol` is an advanced wire-level package and is not the preferred stable integration surface.
- Raw packet/platform backend/SYN-sequence APIs in `knock` are experimental for the v0.1.x release-candidate line.
- `cmd/knock-proxy` is a relay compatibility command, not the full historical product and not a unified CLI for every SDK gate mode.

## Dependency and archive model

- Source release archives use Go modules and do not include `vendor/`.
- Keep `go.work` and `go.work.sum`; they bind the repository's root, examples, observability, and integration modules for local development and CI.
- Offline builds require a populated module cache or dependency mirror.

## Breaking changes

- UDP JSON knock packets are removed. UDP knock now requires binary AEAD frames.
- No JSON fallback is provided.
- New clients and servers must agree on binary UDP knock framing.
- Legacy JSON fields are rejected only for knock-frame compatibility settings; unrelated JSON output/logging settings remain valid.
- Root package APIs are narrowed; advanced knobs are available through subpackages.

## Validation status

Automated validation covers unit tests, race smoke tests, vet, fuzz smoke, dry-run firewall command checks, localhost integration, nested Prometheus/gRPC modules, example builds, release archive checks, and cross-platform compilation.

Not fully validated by repository automation:

- Linux nftables / iptables / ipset-iptables real host enforcement.
- UDP passive DROP behavior.
- Windows WinDivert / Npcap paths.
- macOS BPF / pcap paths.
- Long-duration fuzzing and production performance characterization.
