# v0.1.0-rc2.4 release notes

rc2.4 hardens firewall lifecycle semantics and finishes the release packaging/documentation pass for the rc2 line.

## Highlights

- Fixed knock frame benchmark smoke coverage after replay-cache hardening by building fresh frames per iteration with an explicit replay cache.
- Hardened firewall cleanup, rollback, revoke, and `IsAllowed` semantics across nftables, iptables, and ipset-iptables.
- Added default `allow_seconds` handling in the firewall SDK path and explicit `enable_ipv6: false` support for IPv4-only iptables/ipset deployments.
- Guarded nftables cleanup against dangerous system table names; nftables cleanup must use libknock-owned tables.
- Improved `knock-proxy` firewall runtime/doctor summaries so `noop` is visible as non-port-hiding and non-rule-installing.
- Split standard and vendored source archive packaging in `scripts/package-release.sh` and documented offline `-mod=vendor` validation.
- Expanded validation matrix, known limitations, API-surface guidance, LLM recipes, examples index, and release checklist.

## Artifacts

`with-vendor` remains a dedicated offline/reproducibility artifact. The normal source archive excludes `vendor/`; the `*-with-vendor.zip` archive contains `vendor/modules.txt`, `go.work`, and `go.work.sum` and should be validated with `-mod=vendor`.

## Known limits

- Firewall behavior is covered by unit, dry-run, and command-sequence tests in this release round; target-host privileged validation is still required for production nftables/iptables/ipset deployments.
- Windows/macOS passive capture paths and OpenWrt/container firewall behavior remain target-environment validation items, not broad production claims.
