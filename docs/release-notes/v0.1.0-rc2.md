# v0.1.0-rc2 release notes

`libknock` v0.1.0-rc2 is a GitHub pre-release candidate focused on shutdown observability, startup validation, and release-process hardening after rc1 review.

## Highlights

- Added `libknock.NewListener` / `WrapListenerE` so production startup paths can receive listener configuration errors directly. `WrapListener` remains as the convenience `net.Listener` API where errors surface from `Accept`.
- Firewall-backed gate cleanup now records cleanup/revoke failures, emits firewall-error events, and returns aggregated cleanup errors from `Gate.Close`.
- Listener shutdown now cancels the auth context used by in-flight workers; per-connection auth still observes `AuthTimeout`.
- `Dialer.DialContext` validates client auth configuration before knock or TCP network side effects.
- SYN replay cache now has its own lock and atomic add-if-absent semantics, with unambiguous hex nonce replay keys.
- Server-proof nonce/prefix checks use the same constant-time helper style as proof-tag checks.
- Policy rate-limit fallback keys distinguish nil, TCP, split-host-port, and custom `net.Addr` forms.
- Release scripts now invoke sub-scripts through `bash`; strict release mode fails if `dupl` is unavailable.
- License and release-gate documentation now distinguish source-available BUSL terms, CI checks, and local release gates more explicitly.


## Security and correctness fixes since rc2.1

- Replay caches now fail closed at capacity: expired entries are swept first, and active nonces are never evicted to admit a new nonce.
- Knock frame server parsing now requires a replay cache through `OpenKnockFrame`; the unsafe no-replay parser is explicitly named `ParseKnockFrameUnsafe`.
- `AuthenticatedListener` per-connection auth timeout cancellation is scoped to the single authentication attempt instead of the worker loop.
- Rate limiters now reject new high-cardinality keys when all buckets are active instead of evicting active buckets.
- Short-TTL bans now use a TTL-derived sweep interval, reducing stale stored entries and metric skew.
- `TTLLRU.Len()` is documented as a stored-entry upper bound, with `ActiveLen(now)` available for exact active counts.
- Envelope v2 server-proof comments and tests clarify that the proof echoes 16 bytes but HMAC-binds the full 24-byte prefix.
- `ServerConfig.Validate()` now normalizes defaults internally, matching the envelope config style.
- `KnockSessionStore` documents its lock order around nested cache access.
- Script firewall backends validate configured commands during `Init()` without executing firewall side effects.

## Validation status

Automated validation covers unit tests, vet, build, race smoke, fuzz smoke, benchmark smoke, documentation link checks, duplication checks, nested modules, example builds, and release archive checks when `scripts/release-check.sh` and `scripts/package-release.sh` pass.

The same target-host caveats from rc1 still apply: nftables/iptables/ipset enforcement, UDP passive capture, TCP SYN capture, Windows packet integrations, macOS packet integrations, long-running fuzz, and production performance characterization require target environment validation before production claims.
