# v0.1.0-rc2.5 release notes

## Highlights

- Hardens the release gate so vendored validation runs in a temporary checkout instead of deleting an existing `vendor/` tree from the working copy.
- Aligns the optional Prometheus module with the `v0.1.0-rc2.5` parent module dependency for pre-release consumers and normalizes nested gRPC example protobuf versions for workspace vendor consistency.
- Adds API-surface snapshot checking for stable root-package exports.
- Adds longer fuzz and benchmark helper scripts for protocol, knock, auth, replay-cache, sequence-tracker, and gate paths.
- Clarifies envelope v2 bucket sizing relative to server `MaxFrameSize`.
- Strengthens production documentation for iptables fallback cleanup risk, passive UDP privileges, firewall validation, artifact audit, and target-host validation boundaries.

## Validation run for this tag

Before publishing this tag, run at minimum:

```sh
scripts/check.sh
scripts/check-api.sh
scripts/release-check.sh
scripts/package-release.sh --with-vendor v0.1.0-rc2.5 dist/
```

Optional longer pre-stable evidence:

```sh
FUZZTIME=10m scripts/fuzz-long.sh
BENCHTIME=3s BENCHCOUNT=3 scripts/benchmark.sh
```

## Artifacts

Publish both source archives:

- `libknock-v0.1.0-rc2.5.zip` for ordinary Go module users.
- `libknock-v0.1.0-rc2.5-with-vendor.zip` for offline review, reproducible local audit, LLM-assisted integration, and restricted CI.

Each archive should have a corresponding `.sha256` file. The standard archive must not include `vendor/`; the `with-vendor` archive must include `vendor/modules.txt` and pass `-mod=vendor` validation.

## Known limits

This RC still does not claim full-machine validation for every firewall or packet-capture backend. Treat these as target-host validation items before production claims:

- Linux nftables, ipset-iptables, and iptables real rule creation, revoke, cleanup, repeated startup, and crash-recovery cleanup.
- `iptables` fallback timeout semantics; plain iptables has process-managed expiry and can leave ACCEPT rules after an unclean exit until cleanup runs.
- UDP passive capture with `drop_udp_knock_port`, including root/CAP_NET_RAW/pcap privileges and firewall DROP interaction.
- TCP SYN and SYN sequence paths.
- Windows WinDivert/Npcap paths.
- macOS raw/BPF/pcap paths.
- Long-running fuzz and performance characterization beyond the recorded release run.
